# PuuNote AI Models Configuration


Это руководство для LLM моделей (AI-ассистентов) по поведению приложения в отношении выбора и использования моделей Gemini.

## Основной принцип (Fallback Chain)

Приложение позволяет отправлять запросы в нейросети через плагины (например, Voice Fixer, PuuExtend и др.). Так как разные модели могут отклонять запросы из-за лимитов (Quota/Rate Limits), либо API-ключи пользователей могут иметь ограничения, приложение по умолчанию использует каскадный механизм перебора ключей и моделей.

Все ИИ-запросы должны выполняться через функцию `generateContentFallback(prompt, customModelOverride)` из файла `src/utils/aiModels.ts`, а не напрямую через инициализацию `GoogleGenAI`.

## Модели по умолчанию (DEFAULT_MODELS)

По умолчанию приложение использует следующий порядок прохождения (от самых мощных/тяжелых к более простым и быстрым):
`gemini-2.5-pro, gemini-3-flash-preview, gemini-2.5-flash, gemini-3.1-flash-lite`

### Обоснование выбора:
1. `gemini-2.5-pro`: Передовая флагманская модель для сложных рассуждений и написания текста. Остается в приоритете как самая сильная для бесплатных лимитов.
2. `gemini-3-flash-preview`: Интеллектуальная и быстрая модель 3-го поколения с улучшенными функциями поиска.
3. `gemini-2.5-flash`: Быстрая, надежная модель с огромным контекстом (до 1 млн токенов), идеально подходит как запасной вариант.
4. `gemini-3.1-flash-lite`: Самая компактная, экономичная и быстродействующая модель. Отличный замыкающий вариант, если все остальные лимиты превышены.

**Внимание**: `gemini-3.1-pro` в списке отсутствует намерено, так как у нее нет бесплатного тарифа (Free Tier).

## Использование и изменение конфигурации

Если пользователь просит добавить новую модель или изменить приоритеты:
1. Окно глобальных настроек плагинов (в `PluginsPanel.tsx`) позволяет пользователю визуально редактировать этот список через запятую.
2. Значение списка сохраняется в `localStorage` под ключом `GLOBAL_GEMINI_MODELS`.
3. Системные значения по умолчанию хранятся в константе `DEFAULT_MODELS` в `src/utils/aiModels.ts`. Если нужно программно изменить дефолтный список, редактировать нужно именно эту константу.

## Конвенции UI

Всплывающие окна настроек, содержащие поля ввода (теги `input`, `textarea`), должны быть оформлены единообразно. Для фона `textarea` и `input` внутри модальных окон карточек необходимо использовать `bg-app-input-bg`. **Никогда** не используйте `bg-app-bg` или `bg-app-card` для полей ввода. Класс `bg-app-input-bg` (переменная `--app-input-bg`) гарантирует:
- Некоторое отличие в Светлой и Темной темах, где он всегда светлее фона плашки/модального окна (или белый).
Также используйте `focus:ring-1 focus:ring-inset focus:ring-app-accent` для корректного отображения рамки фокуса внутри элемента.
